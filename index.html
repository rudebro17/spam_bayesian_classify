01 / The Concept

What is a Naive Bayes Classifier?

📊

Probability at its Core

Naive Bayes is a probabilistic machine learning algorithm rooted in Bayes' Theorem. Instead of drawing a line between data points, it asks: "Given the words in this message, how likely is it to be spam?"

📚

Learns from Examples

The model trains on labelled messages (spam vs. ham) and builds a vocabulary frequency table. Words like "win", "free", and "prize" appear overwhelmingly more in spam — the model captures that pattern.

⚡

Why "Naive"?

It assumes each word in the message is conditionally independent of the others — a simplification that is rarely true in real English, but works surprisingly well in practice for text classification tasks.

🛡️

Laplace Smoothing

When a word in the test message never appeared during training, a raw probability of 0 would break everything. We add +1 to every word count (Laplace / add-one smoothing), ensuring no word is impossible.

04 / The Mathematics

Bayes' Theorem Explained

Bayes' Theorem
P(Spam | Words) ∝ P(Spam) × ∏ P(wᵢ | Spam)

The probability that a message is spam, given its words, is proportional to the prior probability of spam multiplied by the product of the likelihood of each word appearing in a spam message.

Log Probability (used in code)
log P(Spam|W) = log P(Spam) + Σ log P(wᵢ|Spam)

We use logarithms instead of multiplication because multiplying thousands of tiny probabilities causes underflow (becomes 0). Summing logs avoids this numerical stability issue.

Laplace Smoothing
P(wᵢ | spam) = (count(wᵢ in spam) + 1) / (total spam words + |vocab|)

Adding +1 to every word count ensures that a word never seen in training does not produce log(0) = −∞, which would destroy the entire calculation.

Decision Rule
class = argmax{ log P(Spam|W), log P(Ham|W) }

We compute the log-probability for both classes and simply return whichever is higher — no threshold needed.

06 / About

About This Project

This project is a from-scratch implementation of the Naive Bayes text classifier — no scikit-learn, no NLTK, no external ML libraries. Every element of the algorithm, from the preprocessing pipeline to the Laplace-smoothed Bayesian inference, was written in pure Python.

The model trains on a custom dataset of 80 messages, split 80/20 into training and test sets. Despite its simplicity, it achieves ~90 % accuracy — demonstrating how effective probabilistic methods are for text classification.

This is an ideal exhibit for understanding the foundations of Natural Language Processing (NLP) and machine learning without the black-box abstraction of high-level frameworks.
